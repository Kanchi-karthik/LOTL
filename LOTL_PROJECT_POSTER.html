LOTLGuard: Living-Off-The-Land Threat Detection Framework

ML-Driven Real-Time SOC Dashboard with MITRE Mapping, Quarantine, and Incident Response

Scan Repo

Objective

  • Detect LOTL command misuse in real time with behavioral and anomaly signals.
  • Correlate events with threat scoring and MITRE ATT&CK mapping.
  • Provide SOC dashboard for incidents, topology, and attacker geolocation.
  • Trigger automated response with quarantine and forensic capture.

Core Modules

  • Dashboard: Flask UI, APIs, live charts, incident timeline.
  • Backend: parsing, enrichment, threat scoring, persistence.
  • Detection: Isolation Forest with command feature extraction.
  • Response: severity-based containment and auto-block logic.

Implementation Highlights

  • Python 3.8+, Flask, scikit-learn, joblib stack.
  • Features include entropy, risky tools, paths, network indicators.
  • Persistent quarantine logs and forensic snapshots.
  • Falco-compatible ingestion with Kubernetes monitoring hooks.

Results

  • Model accuracy around 95% on controlled test data.
  • False positives tuned near 2% for balanced alerting.
  • Detection latency under 50 ms per prediction.
  • Threat score normalized to 0 to 100 for SOC triage.
  • Automated responses across low to critical severity.

Conclusion

  • LOTLGuard delivers practical and reproducible LOTL threat detection.
  • Combines ML, rules, and response orchestration in one workflow.
  • Designed for clear SOC visibility and rapid incident containment.
  • Ready for presentation and demo screenshots.
